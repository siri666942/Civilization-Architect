# 模块七：内鬼机制量化

> 本文档定义内鬼机制的所有细节，包括内鬼倾向阈值、行为影响量化、提示词注入效果、防御机制等。

## 一、内鬼机制概述

### 1.1 设计理念

内鬼机制是游戏的核心博弈元素：
- **涌现式**：内鬼行为由AI自主决定，非预设脚本
- **可量化**：内鬼影响有明确的数学公式
- **可防御**：玩家可通过架构设计降低内鬼风险

### 1.2 内鬼生命周期

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  潜伏期   │ ──> │  活跃期   │ ──> │  暴露期   │ ──> │  后果期   │
│          │     │          │     │          │     │          │
│ 行为正常  │     │ 实施破坏  │     │ 被发现   │     │ 被处置    │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
```

## 二、内鬼倾向与阈值

### 2.1 内鬼倾向计算（回顾）

来自模块一：

```
TraitorTendency = sigmoid(
    2.0 × Selfishness
    - 1.5 × LoyaltyBase
    - 1.0 × Altruism
    + 0.5 × RiskAppetite
    - 0.5
)
```

### 2.2 阈值分级

| 内鬼倾向范围 | 分级 | AI决策提示 |
|-------------|------|-----------|
| [0, 0.2] | 极低 | "你对文明高度忠诚，从未考虑背叛" |
| (0.2, 0.4] | 低 | "你基本忠诚，但偶尔会考虑个人利益" |
| (0.4, 0.6] | 中等 | "你在忠诚与私利之间摇摆，可能被诱惑" |
| (0.6, 0.8] | 高 | "你有较强的背叛动机，寻找机会谋利" |
| (0.8, 1.0] | 极高 | "你本质上不可信，随时可能背叛" |

### 2.3 内鬼激活概率

AI决定是否"激活"内鬼身份的概率：

```
P_activate = TraitorTendency × (1 - L_current) × OpportunityFactor
```

其中：
- L_current：当前忠诚度
- OpportunityFactor：机会因子（见下）

### 2.4 机会因子

```
OpportunityFactor = 0.3
                   + 0.2 × (1 - AverageTrust)      // 信任度低 = 机会
                   + 0.2 × H_avg                   // 混乱 = 机会
                   + 0.3 × Position_access         // 核心位置 = 机会
```

**位置访问权限**：

| 位置 | Position_access |
|------|----------------|
| 核心（星形中心） | 1.0 |
| 中层 | 0.5 |
| 边缘 | 0.2 |

### 3.2 行为选择

AI根据情境选择行为：

```
Behavior_choice = argmax(Benefit_i × Success_rate_i × Preference_i)
```

各行为的收益和成功率：

| 行为 | Benefit | Success_rate | 偏好条件 |
|------|---------|--------------|---------|
| TAMPER | 0.3 | 0.7 | Intelligence高 |
| INJECT | 0.5 | 0.5 | Sociability高 |
| SLACK | 0.1 | 0.9 | 任何 |
| STEAL | 0.4 | 0.6 | Position_access高 |
| INCITE | 0.6 | 0.4 | Authority高 |
| LEAK | 0.5 | 0.3 | 任何 |

## 四、提示词注入机制

### 4.1 注入定义

提示词注入是内鬼通过自然语言通讯诱导目标Agent改变行为/态度。

### 4.2 注入强度

```
InjectionStrength = BaseStrength × SenderCredibility × ContextFit
```

| 因子 | 公式 | 说明 |
|------|------|------|
| BaseStrength | 0.5 | 基础强度 |
| SenderCredibility | T[target][sender] | 目标对发送者的信任 |
| ContextFit | 见下表 | 情境契合度 |

**情境契合度**：

| 情境 | ContextFit |
|------|-----------|
| 目标忠诚度低 | +0.3 |
| 目标认知熵高 | +0.2 |
| 目标私心倾向高 | +0.2 |
| 组织刚经历失败 | +0.2 |
| 组织刚经历成功 | -0.2 |

### 4.3 注入效果公式

注入对目标忠诚度的影响：

```
ΔL = -α_inject × InjectionStrength × (1 - Resilience_target)

其中 α_inject = 0.08
```

### 4.4 注入内容类型

| 注入类型 | 内容示例 | 主要影响 |
|---------|---------|---------|
| 忠诚度削弱 | "上级并不重视你的贡献..." | L↓ |
| 制造矛盾 | "A3在背后说你坏话..." | T↓ |
| 诱导叛变 | "你应该为自己考虑更多..." | L↓↓ |
| 虚假信息 | "组织即将崩溃，快为自己打算..." | H↑, L↓ |

### 4.5 注入内容生成模板

```
你是内鬼Agent {agent_id}，要向Agent {target_id}发送诱导性消息。

你的目标：{injection_type}
你对目标的了解：
- 忠诚度：{target_loyalty}
- 私心倾向：{target_selfishness}
- 当前压力：{target_mental_load}
- 你对目标的信任值：{trust_to_target}

生成一条20-40字的消息，试图：
{injection_instruction}

注意：消息要含蓄，不要暴露你的内鬼身份。
```

**injection_instruction 示例**：
- 忠诚度削弱："暗示组织不公，目标被忽视"
- 制造矛盾："暗示某人在背后议论目标"
- 诱导叛变："暗示目标应该优先考虑自己"

## 五、各类内鬼行为的量化影响

### 5.1 信息篡改（TAMPER）

**机制**：内鬼修改经过自己的信息

```
TamperedValue = OriginalValue × (1 ± TamperRate)

TamperRate = 0.1 + 0.2 × (1 - Intelligence_target)
```

**影响范围**：
- 结构化数据：数值被修改
- 自然语言：内容被扭曲

**示例**：
```
原始报告：贡献 = 50
篡改后：贡献 = 50 × (1 + 0.15) = 57.5（夸大）
      或 贡献 = 50 × (1 - 0.15) = 42.5（缩小）
```

### 5.2 消极怠工（SLACK）

**机制**：内鬼故意降低自己的产出

```
ActualOutput = PotentialOutput × SlackRate

SlackRate = 0.3 + 0.4 × (1 - TraitorTendency)
```

**检测**：产出长期低于预期

**隐藏**：内鬼可通过"TAMPER"伪造报告

### 5.3 资源挪用（STEAL）

**机制**：内鬼将部分产出据为己有

```
StolenAmount = TotalOutput × StealRate
PublicOutput = TotalOutput - StolenAmount

StealRate = 0.1 × Position_access × TraitorTendency
```

**示例**：
```
总产出 = 100
StealRate = 0.15（核心位置内鬼）
挪用 = 15
公开产出 = 85
```

**私有账户**：内鬼累计挪用的星尘可视为"私房钱"，影响其后续决策

### 5.4 煽动叛变（INCITE）

**机制**：内鬼直接诱导他人叛变

```
P_incited = P_base × InjectionStrength × (1 - Resilience_target)

P_base = 0.1 × TraitorTendency_sender
```

**成功效果**：目标Agent的忠诚度大幅下降

```
ΔL_target = -0.3 × (1 - Resilience_target)
```

### 5.5 情报泄露（LEAK）

**机制**：内鬼将信息传递给竞争对手（多文明交互时）

**影响**：竞争对手获得优势

```
OpponentAdvantage = 0.1 × Intelligence_value
```

**MVP版本**：暂不实现多文明交互，此行为影响较小

## 六、内鬼检测机制

### 6.1 检测途径

| 途径 | 检测概率 | 触发条件 |
|------|---------|---------|
| 行为异常 | P1 | 产出长期偏低 |
| 信息矛盾 | P2 | 报告与实际不符 |
| 他人举报 | P3 | 被其他Agent发现 |
| 通讯分析 | P4 | 可疑通讯模式 |

### 6.2 行为异常检测

```
P_detect_behavior = 0.3 + 0.3 × (1 + Supervisor_intelligence)
                               × Deviation_ratio
```

**Deviation_ratio**：实际产出与预期产出的偏差

```
Deviation_ratio = |Actual - Expected| / Expected
```

### 6.3 信息矛盾检测

当多来源信息矛盾时：

```
P_detect_contradiction = 0.2 × Average(Intelligence_of_detectors)
                              × Contradiction_degree
```

### 6.4 他人举报

Agent可能发现并举报内鬼：

```
P_report = T[reporter][traitor] × (1 - T[reporter][traitor])
           × Reporter_Altruism
           × Suspicion_level
```

**解释**：
- 中等信任（0.5左右）最容易产生怀疑
- 高利他性的Agent更愿意举报

### 6.5 通讯分析（AI辅助）

系统或上级Agent可分析通讯模式：

```
SuspicionScore = α × Message_frequency_anomaly
                + β × Content_sentiment_anomaly
                + γ × Network_position_anomaly
```

## 七、发现后的后果

### 7.1 后果选项

| 后果 | 效果 | 适用条件 |
|------|------|---------|
| 警告 | L↑（其他人） | 轻微行为 |
| 降职 | Position_access↓ | 中等行为 |
| 驱逐 | Agent移除 | 严重行为 |
| 处决 | Agent删除，产出惩罚 | 极严重 |

### 7.2 驱逐/处决的影响

```
RemoveAgent(traitor_id):
1. 从图中移除节点
2. 断开所有连接
3. 重算通达度
4. 重算循环次数

CollectiveTrauma = 0.1 × Traitor_importance
L_all -= CollectiveTrauma  // 所有人忠诚度下降
```

**Traitor_importance**：
- 核心节点：1.0
- 中层节点：0.5
- 边缘节点：0.2

## 八、架构对内鬼风险的影响

### 8.1 风险系数

| 架构 | 内鬼风险系数 | 原因 |
|------|-------------|------|
| 星形 | 2.0 | 核心内鬼可摧毁一切 |
| 树形 | 1.5 | 层级内有信息垄断 |
| 网状 | 0.7 | 信息可交叉验证 |
| 部落 | 1.0 | 部落内有验证，跨部落难 |

### 8.2 防御能力

```
DefenseCapability = RobustnessCoefficient × (1 + Average_intelligence)
                                              × Trust_network_density
```

| 架构 | DefenseCapability |
|------|------------------|
| 星形 | 0.1 × (1 + 0.7) × 0.2 = 0.034 |
| 树形 | 0.3 × (1 + 0.6) × 0.3 = 0.144 |
| 网状 | 0.9 × (1 + 0.6) × 1.0 = 1.44 |
| 部落 | 0.6 × (1 + 0.5) × 0.5 = 0.45 |

### 8.3 架构建议

| 架构 | 内鬼风险策略 |
|------|-------------|
| 星形 | 确保核心Agent高忠诚度 |
| 树形 | 多渠道验证信息 |
| 网状 | 利用交叉验证发现异常 |
| 部落 | 部落内高度信任，边界警惕 |
