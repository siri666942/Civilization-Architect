# 模块四：通讯与信息传播系统

> 本文档定义Agent间通讯的机制，包括结构化数据、自然语言文本、信息失真公式、通讯收益量化。

---

## 一、通讯系统概述

### 1.1 通讯的双轨制

Agent通讯包含两种形式：

| 类型 | 性质 | 用途 | 示例 |
|------|------|------|------|
| 结构化数据 | 定量、精确 | 状态汇报、任务分配 | "产出：35星尘" |
| 自然语言文本 | 定性、灵活 | 情感表达、劝说、诱导 | "我们需要更多合作..." |

### 1.2 通讯流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Agent i    │     │   通讯通道   │     │  Agent j    │
│  发送方     │ ──> │  （失真）    │ ──> │  接收方     │
│             │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       │                   │                   │
   选择内容            应用失真            解析处理
   消耗体力            加噪声              更新状态
```

## 二、结构化数据通讯

### 2.1 数据类型定义

| 数据类型 | 字段 | 格式 | 用途 |
|---------|------|------|------|
| 工作汇报 | report | JSON | 向上级汇报工作进展 |
| 任务分配 | task | JSON | 上级向下级分配任务 |
| 资源请求 | request | JSON | 请求资源或帮助 |

| 投票表决 | vote | JSON | 集体决策 |

```json
    "issues": ["需要更多资源"],
    "sentiment": 0.6           // 情绪状态 [0,1]
  }
}
```

### 2.4 状态同步格式

## 三、自然语言文本通讯

### 3.1 生成机制

自然语言文本由LLM根据以下输入生成：
- Agent的性格属性
- 当前状态（认知熵、忠诚度等）
- 通讯目的
- 与接收方的关系（信任值）

### 3.2 AI生成提示词模板

```
你是Agent {agent_id}，你的性格特点：
- 权威感度：{authority}
- 私心倾向：{selfishness}
- 韧性：{resilience}
- 利他性：{altruism}
- 社交性：{sociability}
- 风险偏好：{risk_appetite}
- 智力：{intelligence}
- 忠诚度基准：{loyalty_base}

你当前的状态：
- 剩余体力：{energy}%
- 认知熵：{cognitive_entropy}
- 当前忠诚度：{loyalty}
- 心理负担：{mental_load}

你要向Agent {receiver_id}发送消息。
你对TA的信任度：{trust_to_receiver}
你们的关系：{relationship_description}

通讯目的：{communication_purpose}

请生成一条简短的消息（20-50字），表达你的意图。
输出JSON格式：
{
  "message": "你的消息内容",
  "tone": "friendly|neutral|hostile|manipulative",
  "hidden_intent": "你可能有隐藏意图（如有）"
}
```

### 3.3 通讯目的分类

| 目的 | 频率权重 | 典型场景 |
|------|---------|---------|
| 汇报 | 30% | 向上级报告进展 |
| 请求 | 20% | 请求资源或帮助 |
| 协作 | 25% | 同级协调工作 |
| 社交 | 15% | 建立或维护关系 |
| 影响力 | 10% | 试图说服或诱导 |

### 3.4 消息长度与体力的关系

长消息消耗更多体力：

```
E_comm = BaseCost + LengthCost × word_count
```

其中：
- BaseCost = 5
- LengthCost = 0.1
- word_count 建议 20-50 字

## 四、信息失真机制

### 4.1 失真来源

信息在传播过程中会失真，失真来源包括：

| 失真来源 | 影响 | 理论依据 |
|---------|------|---------|
| 架构层级 | 每经过一层，信息衰减 | 电话游戏效应 |
| 信任偏差 | 低信任时信息被削弱 | 选择性注意 |
| 认知熵 | 高熵时理解偏差 | 信息处理能力下降 |
| 恶意篡改 | 内鬼故意扭曲 | 信息战 |

### 4.2 单步传播失真公式

信息从 i 传到 j 的失真系数：

```
Distortion(i→j) = D_trust × D_relation × D_entropy
```

#### 4.2.1 信任失真

```
D_trust = 0.5 + 0.5 × T[j][i]
```

- T[j][i]=1 时，D_trist=1（无失真）
- T[j][i]=0.5 时，D_trust=0.75（25%失真）
- T[j][i]=0 时，D_trust=0.5（50%失真）

#### 4.2.2 关系失真

```
D_relation = 1 - 0.1 × |level(i) - level(j)|
```

同级通讯失真少，跨层级通讯失真多。

#### 4.2.3 认知熵失真

```
D_entropy = 1 - 0.3 × H_j
```

接收方认知熵高，理解有偏差。

### 4.3 多步传播累积失真

信息经过 k 步传播，从 i 到 j：

```
Distortion_total = Π Distortion_step_m
```

**示例**：A1 → A2 → A3 → A4

```
假设每步失真系数为 0.8
Distortion_total = 0.8 × 0.8 × 0.8 = 0.512
```

最终信息保真度 = 51.2%

### 4.4 结构化数据失真

对于数值型数据，失真表现为随机误差：

```
ReceivedValue = SentValue × Distortion × (1 + Noise)
```

其中 Noise ∈ N(0, σ²)，σ = 0.05

**示例**：
```
发送值 = 50（星尘贡献）
失真系数 = 0.8
噪声 = -0.02

接收值 = 50 × 0.8 × (1 - 0.02) = 39.2
```

### 4.5 自然语言失真

自然语言失真表现为语义偏移：

| 失真程度 | 偏移示例 |
|---------|---------|
| 轻微(>0.8) | "我们需要合作" → "我们要协作" |
| 中等(0.5-0.8) | "我们应该全力投入" → "我们需要多投入一些" |
| 严重(<0.5) | "必须立即行动" → "也许可以等等看" |

**LLM处理方式**：在转发时，添加失真提示

```
你收到的信息经过了{hop_count}层传播，可能存在失真。
原始信息："{original_message}"
请根据你的理解，重新表述这条信息（可能有所偏差）。
```

## 五、通讯收益量化

### 5.1 个体收益

通讯对发送方的收益：

```
IndividualBenefit = Benefit_information + Benefit_social + Benefit_political
```

| 收益类型 | 计算公式 | 说明 |
|---------|---------|------|
| 信息收益 | +0.05 × 新获得有用信息数 | 获取有价值信息 |
| 社交收益 | +0.02 × Trust_increase | 建立关系 |
| 政治收益 | +0.03 × Authority_effect | 向上管理收益 |

**Authority_effect**：权威感度高的Agent更善于向上管理

```
Authority_effect = Authority × Response_quality
```

### 5.2 集体收益

通讯对文明的整体收益：

```
CollectiveBenefit = Benefit_coordination + Benefit_knowledge_sharing
```

| 收益类型 | 计算公式 | 说明 |
|---------|---------|------|
| 协调收益 | +0.02 × synchronized_agents | 同步协作的Agent数 |
| 知识共享 | +0.01 × information_spread | 信息传播范围 |

### 5.3 信息保真度的宏观影响

文明整体信息保真度：

```
Fidelity_global = Average(Distortion_actual) for all communications
```

信息保真度影响最终产出。

## 六、通讯频率控制

### 6.1 通讯触发条件

| 条件 | 触发概率 | 说明 |
|------|---------|------|
| 常规汇报 | 100%/循环 | 向直接上级汇报 |
| 异常事件 | 触发立即 | 发现问题立即通讯 |
| 请求帮助 | 按需 | 遇到困难时 |
| 社交维护 | P=0.2/循环 | 随机社交 |

### 6.2 通讯负担

过多通讯导致信息过载：

```
InfoLoad_i = Σ (MessageCount_j × Importance_j) / ProcessingCapacity_i

where:
ProcessingCapacity_i = Intelligence × (1 - H_i) × 10
```

当 InfoLoad > 1 时：
- Agent无暇处理所有信息
- 可能忽略部分消息
- 认知熵增加：ΔH += 0.05 × (InfoLoad - 1)

## 七、内鬼通讯的特殊机制

### 7.1 提示词注入通讯

内鬼通过自然语言文本注入诱导性信息：

```json
{
  "type": "manipulation",
  "sender": "A7",  // 内鬼Agent
  "receiver": "A3",
  "content": {
    "message": "上级其实并不重视你的贡献，你应该多为自己考虑...",
    "hidden_intent": "降低忠诚度，诱导叛变",
    "manipulation_strength": 0.7
  }
}
```

### 7.2 注入效果公式

被注入Agent的忠诚度变化：

```
ΔL = -0.05 × ManipulationStrength × (1 - Resilience)
```

韧性高的Agent更不容易被诱导。

### 7.3 识别机制

被注入Agent有一定概率识别出诱导：

```
DetectionProbability = Intelligence × H_factor × Context_clarity

where:
H_factor = 1 - 0.5 × H  // 认知熵低则识别能力强
Context_clarity = 收到的相关警告数 × 0.1
```
