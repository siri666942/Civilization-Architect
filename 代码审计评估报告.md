## 二、严重问题 (Critical)

### 2.1 🔴 导入路径不一致

**问题描述**: 代码使用相对导入，但模块结构不支持直接运行。

**受影响文件**:
- `core/god_agent.py:12` - `from models.agent import ...`
- `core/engine.py:12-18` - 多处相对导入
- `core/macro_variables.py:10-11` - 相对导入

**示例**:
```python
# core/god_agent.py:12
from models.agent import Agent, Personality, AgentState  # 依赖运行时路径
```

**风险**: 直接运行单个文件会报 ModuleNotFoundError

**建议**:
1. 使用绝对导入: `from backend.models.agent import ...`
2. 或添加 `__init__.py` 的正确导出配置
3. 使用 `python -m backend.tests.simulation_runner` 方式运行

---

### 2.2 🔴 随机数使用不一致

**问题描述**: `engine.py` 中使用标准库 `random`，而其他地方使用 `numpy.random`。

**受影响位置**: `engine.py:300, 310, 320, 348`

```python
# engine.py:300
import random  # 在函数内部导入
if random.random() < p_activate * 0.5:  # 不受seed控制
```

**风险**: 无法保证结果可复现

**建议**: 统一使用 `numpy.random` 或在 GameEngine 中维护 `rng` 实例

---

### 2.3 🔴 内鬼激活阈值硬编码

**问题描述**: 内鬼激活使用固定阈值和魔法数字。

**受影响位置**: `engine.py:304-311`

```python
if agent.traitor_tendency > 0.4 and not agent.is_active_traitor:  # 硬编码阈值
    opportunity = 0.3 + 0.2 * (1 - agent.get_avg_trust()) + 0.2 * agent.state.cognitive_entropy
    p_activate = agent.traitor_tendency * (1 - agent.state.loyalty) * opportunity
    if random.random() < p_activate * 0.5:  # 硬编码系数
```

**风险**: 参数调优困难，无法通过配置调整

**建议**: 提取为配置参数或 TraitorConfig 类

## 三、中等问题 (Medium)

### 3.1 🟡 树形架构硬编码节点数

**问题描述**: 树形架构假设固定10个Agent。

**受影响位置**: `architecture.py:67-112`

```python
def create_tree_architecture(agent_ids: List[str]) -> ArchitectureConfig:
    # A2 -> A5, A6 (硬编码索引)
    adj[1, 4] = 1
    adj[1, 5] = 1
    # ... 更多硬编码
```

**建议**: 实现动态树形结构生成算法

### 3.2 🟡 部落架构固定部落划分

**问题描述**: 部落架构假设固定10个Agent和固定部落分配。

**受影响位置**: `architecture.py:137-180`

```python
# 部落1: A1-A4 (索引0-3) - 硬编码

    for j in range(4):
        if i != j:
            adj[i, j] = 1
```

**建议**: 根据Agent数量动态划分部落

### 3.3 🟡 Floyd-Warshall算法效率问题

**问题描述**: 每次分析架构都重新计算最短路径，时间复杂度 O(n³)。

**受影响位置**: `architecture.py:200-222`

```python
def compute_shortest_paths(adj: np.ndarray) -> np.ndarray:
    # Floyd-Warshall O(n^3)
    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i, k] + dist[k, j] < dist[i, j]:
                    dist[i, j] = dist[i, k] + dist[k, j]
```

**风险**: 大规模Agent时性能下降明显

**建议**:
1. 缓存计算结果
2. 或使用更高效的算法（如BFS对于无权图）

---

### 3.4 🟡 缺乏类型检查
