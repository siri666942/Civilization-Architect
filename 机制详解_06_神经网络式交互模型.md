# 模块六：神经网络式交互模型

> 本文档将Agent社会建模为神经网络，定义节点状态更新、边权重传播、信息聚合等机制，结合社会网络理论和动力系统理论。

---

## 一、理论基础

### 1.1 社会网络与神经网络的类比

| 神经网络概念 | 社会网络对应 | 游戏中的映射 |
|-------------|-------------|-------------|
| 神经元（Neuron） | 社会个体 | Agent |
| 突触权重（Synaptic Weight） | 社会关系强度 | 信任度 T[i][j] |
| 激活值（Activation） | 个体状态 | 忠诚度、认知熵等 |
| 前向传播 | 信息传递 | Agent通讯 |
| 反向传播 | 社会学习/反馈 | 信任更新 |
| 网络结构 | 组织架构 | 拓扑结构 |
| 损失函数 | 社会压力 | 心理负担 |

### 1.2 动力系统视角

Agent社会可视为**耦合动力系统**：

```
dState/dt = f(State, Input, Parameters)
```

每个Agent的状态随时间演化，受输入和参数影响。

## 二、节点模型（Agent作为神经元）

### 2.1 状态向量

每个Agent在时刻 t 的状态向量为：

```
State_i(t) = [L_i(t), H_i(t), E_i(t), P_i(t)]
```

- L_i：忠诚度
- H_i：认知熵
- E_i：体力值
- P_i：贡献值

### 2.2 参数向量（固定属性）

每个Agent的参数向量为：

```
Param_i = [Authority_i, Selfishness_i, Resilience_i, Altruism_i,
           Sociability_i, RiskAppetite_i, Intelligence_i, LoyaltyBase_i]
```

这些参数由上帝Agent生成，游戏过程中保持不变。

### 2.3 节点激活函数

Agent的"激活值"定义为综合状态指标：

```
Activation_i(t) = w_L × L_i(t) + w_E × (E_i(t)/100) - w_H × H_i(t)
```

其中：
- w_L = 0.4（忠诚度权重）
- w_E = 0.3（体力权重）
- w_H = 0.3（认知熵惩罚权重）

**激活值范围**：[-0.3, 0.7]

**解释**：
- 高激活值：忠诚、有活力、思维清晰
- 低激活值：不忠、疲惫、混乱

## 三、边模型（信任作为权重）

### 3.1 边权重定义

信任矩阵 T 构成网络的权重矩阵：

```
W = T = [T[i][j]]_{N×N}
```

### 3.2 权重的性质

| 性质 | 描述 | 游戏意义 |
|------|------|---------|
| 有向性 | T[i][j] ≠ T[j][i] | 信任不互惠 |
| 动态性 | T(t+1) ≠ T(t) | 信任随交互变化 |
| 有界性 | T ∈ [0, 1] | 信任有上下限 |

### 3.3 权重初始化

初始权重矩阵：

```
T[i][j] = {
  0.5 + 0.1 × (Sociability_i - 0.5)  if j ≠ i
  1.0                                 if j = i
}
```

高社交性的Agent初始信任度略高。

## 四、信息前向传播

### 4.1 单步信息传播

Agent i 向 Agent j 发送信息：

```
Message(i→j) = Content_i × T[i][j]
```

信息被信任度加权。

### 4.2 信息聚合

Agent j 接收来自多个邻居的信息，进行聚合：

```
Input_j = Σ_i Message(i→j) × R_i
        = Σ_i Content_i × T[i][j] × R_i
```

其中 R_i 是信息相关性权重（由架构决定）。

### 4.3 聚合函数类型

可采用不同聚合策略：

| 聚合类型 | 公式 | 适用场景 |
|---------|------|---------|
| 求和 | Σ | 总量累计 |
| 平均 | (1/k)Σ | 平衡各意见 |
| 最大 | max | 取最强信号 |
| 注意力 | Σ × α | 加权平均 |

**注意力机制**（推荐）：

```
α_i = softmax(Content_i × T[i][j]) = exp(Content_i × T[i][j]) / Σ_k exp(Content_k × T[k][j])

Input_j = Σ_i α_i × Content_i
```

高信任来源的信息获得更高权重。

### 4.4 信息传播示例

```
场景：A2、A3、A4 向 A1 发送工作汇报

内容向量（产出贡献）：
Content_2 = 45
Content_3 = 38
Content_4 = 52

信任权重：
T[2][1] = 0.7
T[3][1] = 0.6
T[4][1] = 0.8

注意力权重计算：
score_2 = 45 × 0.7 = 31.5
score_3 = 38 × 0.6 = 22.8
score_4 = 52 × 0.8 = 41.6

α = softmax([31.5, 22.8, 41.6])
α_2 ≈ 0.30
α_3 ≈ 0.22
α_4 ≈ 0.48

聚合输入：
Input_1 = 0.30×45 + 0.22×38 + 0.48×52 = 46.26
```

## 五、状态更新方程

### 5.1 离散时间动力学

Agent状态按离散时间步更新：

```
State_i(t+1) = State_i(t) + ΔState_i
```

### 5.2 忠诚度更新方程

```
L_i(t+1) = L_i(t)
           + α_L × (Input_L - L_i(t))           // 信息输入影响
           - β_L × Conflict_intensity           // 冲突惩罚
           + γ_L × Recognition                  // 认可奖励
           + δ_L × (LoyaltyBase_i - L_i(t))     // 向基准回归
```

参数：
- α_L = 0.1（学习率）
- β_L = 0.05（冲突惩罚系数）
- γ_L = 0.02（认可奖励系数）
- δ_L = 0.05（回归系数）

### 5.3 认知熵更新方程

```
H_i(t+1) = H_i(t)
           + α_H × (InfoLoad_i - Capacity_i)_+  // 过载增加熵
           + β_H × Conflict_count               // 冲突增加熵
           - γ_H × (1 - H_i(t)) × Resilience_i  // 韧性恢复
```

其中 (x)_+ = max(0, x)

参数：
- α_H = 0.03
- β_H = 0.02
- γ_H = 0.05

### 5.4 信任权重更新（类似反向传播）

基于**Hebbian学习规则**和**误差修正**：

```
T[i][j](t+1) = T[i][j](t)
              + η × (Observed_behavior - Expected_behavior)
              + λ × (Activation_i × Activation_j)  // Hebbian项
```

**Hebbian学习**：同时激活的节点之间连接加强

```
Hebbian_term = λ × Activation_i × Activation_j
```

参数：
- η = 0.1（学习率）
- λ = 0.02（Hebbian系数）

### 5.5 完整状态更新矩阵形式

将所有Agent的状态表示为矩阵：

```
State(t) = [L(t), H(t), E(t), P(t)]  // N×4 矩阵

State(t+1) = State(t) + f(State(t), Input(t), Param)
```

其中 Input(t) = T × Content(t)（信息传播）

## 六、网络层次的涌现性质

### 6.1 同步性（Synchronization）

衡量Agent状态的一致程度：

```
Sync = 1 - std(Activation_i) / max_possible_std
```

高同步性 = 所有Agent处于相似状态 = 强凝聚力

### 6.2 扰动传播

单个Agent状态变化对网络的影响：

```
Impact_k = Σ_i ∂Activation_i / ∂State_k
```

通过Jacobi矩阵计算：

```
J = ∂Activation / ∂State = [∂Activation_i / ∂State_j]
```

### 6.3 临界点与相变

网络可能存在临界点：

| 状态 | 特征 | 触发条件 |
|------|------|---------|
| 稳定态 | 扰动后恢复 | 高信任、高韧性 |
| 周期态 | 状态振荡 | 部落架构，部落间竞争 |
| 混沌态 | 微小扰动放大 | 低信任、高熵 |
| 崩溃态 | 网络解体 | 核心节点失效（星形） |

## 七、信息扩散动力学

### 7.1 Bass扩散模型

信息/创新在人群中扩散遵循Bass模型：

```
dI/dt = p × (N - I) + q × (I/N) × (N - I)
```

- I = 已接受信息的Agent数
- p = 创新系数（自发接受）
- q = 模仿系数（受他人影响）

### 7.2 游戏中的应用

谣言、恐慌、忠诚度变化等的传播：

```
dL_affected/dt = p × (N - L_affected) + q × (L_affected/N) × (N - L_affected)
```

高信任网络（q大）：信息传播快
低信任网络（q小）：信息传播慢

## 八、内鬼对网络的影响

### 8.1 内鬼作为噪声源

内鬼发送的虚假信息等效于噪声注入：

```
Content_traitor = Content_true + Noise
Noise ~ N(0, σ_traitor²)
```

σ_traitor 与 ManipulationStrength 成正比。

### 8.2 级联失败模型

内鬼诱导一个Agent叛变，可能触发级联效应：

```
叛变概率：
P_defect(i) = sigmoid(k × (Neighbor_defect_count - threshold))

其中：
threshold = f(Resilience_i, LoyaltyBase_i)
k = 网络敏感度
```

### 8.3 鲁棒性阈值

网络能承受的最大内鬼比例：

```
Critical_traitor_ratio = 1 - 1/<k>
```

其中 <k> 是平均度（连接数）。

- 星形：<k> ≈ 2，临界比例 = 50%
- 网状：<k> = 9，临界比例 = 89%

网状架构能承受更多内鬼。
