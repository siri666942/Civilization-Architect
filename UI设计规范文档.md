# 🎮 文明模拟策略游戏 — UI设计规范文档

## 一、设计系统总览

### 1.1 整体风格定位

**赛博朋克 / 未来科技风格**

- **视觉氛围**：深色背景 + 霓虹光效 + 科技网格纹理
- **核心关键词**：未来感、数据流、全息投影、数字文明
- **情感传达**：神秘、沉浸、策略深度、科技力量感

### 1.2 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| **主背景** | `#0a0e17` → `#111827` 渐变 | 深空黑到暗蓝灰的渐变 |
| **次级背景/卡片** | `#1a1f2e` 半透明 85% | 玻璃拟态卡片底色 |
| **强调色-主** | `#00f0ff` (青色霓虹) | 主要按钮、高亮元素、重要信息 |
| **强调色-辅** | `#ff00ff` (品红霓虹) | 次要操作、警告提示 |
| **强调色-成功** | `#39ff14` (荧光绿) | 正向反馈、数值增长 |
| **强调色-危险** | `#ff3131` (激光红) | 负面反馈、危机警告 |
| **文字-主** | `#e8eaed` | 标题和主要文字 |
| **文字-次** | `#9ca3af` | 辅助说明文字 |
| **边框/分割线** | `rgba(0,240,255,0.2)` | 青色半透明发光边框 |
| **发光效果** | `box-shadow: 0 0 20px rgba(0,240,255,0.3)` | 元素悬浮时的辉光 |

### 1.3 字体系统

| 层级 | 字体 | 大小 | 权重 | 用途 |
|------|------|------|------|------|
| **大标题** | Orbitron / Rajdhani | 36-48px | 700 | 页面标题"文明纪元" |
| **中标题** | Rajdhani / Exo 2 | 24-28px | 600 | 区块标题如"Agent阵列" |
| **小标题** | Exo 2 | 18-20px | 600 | 卡片名称、标签 |
| **正文** | Inter / Source Sans Pro | 14-16px | 400 | 描述文字、聊天内容 |
| **数据/数值** | JetBrains Mono / Fira Code | 16-20px | 500 | 数值显示、变量数据 |
| **标签/徽章** | Rajdhani | 12-14px | 600 | 状态标签、轮次计数 |

### 1.4 通用组件样式

#### 卡片容器

- 背景：深色玻璃效果（`background: rgba(26,31,46,0.85)`）
- 边框：细线发光边框（`border: 1px solid rgba(0,240,255,0.15)`）
- 圆角：8-12px
- 内边距：16-24px
- 悬停效果：边框亮度提升 + 微弱外发光

#### 按钮

**主要按钮（开始游戏/确认）**

- 背景：青色渐变 `linear-gradient(135deg, #00f0ff, #00b4d8)`
- 文字：深色 `#0a0e17`
- 圆角：6px
- 悬停：整体发光增强 + 轻微上浮
- 点击：内缩效果 + 脉冲波纹

**次要按钮（返回/取消）**

- 背景：透明 + 边框 `border: 1px solid rgba(0,240,255,0.5)`
- 文字：青色 `#00f0ff`
- 悬停：背景填充为半透明青色

#### 输入框

- 背景：`rgba(10,14,23,0.8)` 深色填充
- 边框：默认灰色，聚焦时变为青色发光
- 占位符颜色：`#6b7280`
- 聚焦动画：边框从左至右的扫描线效果

## 二、页面1：游戏开始页面

### 2.1 整体布局

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   ┌──────────────────┐    ┌──────────────────┐     │
│   │                  │    │                  │     │
│   │   [区域1]        │    │   [区域2]        │     │
│   │   游戏名称       │    │   用户名输入      │     │
│   │   + 简介         │    │                  │     │
│   │                  │    │   [输入框]       │     │
│   │                  │    │                  │     │
│   │                  │    │   [开始游戏]     │     │
│   └──────────────────┘    └──────────────────┘     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- **布局方式**：左右分栏（桌面端50:50），移动端上下堆叠
- **背景**：动态粒子星空效果 + 底部科技网格线

### 2.2 区域1详情 — 游戏名称与简介

**位置**：页面左侧（或移动端上方）

**游戏名称区域**：

- 名称：「文明纪元」或「星辰文明」（可自定义）
- 字体：Orbitron，48px，青色霓虹带外发光
- 动效：名称字符逐个浮现，带有轻微闪烁
- 下方可选：一行副标题「—— Agent文明的诞生」

**游戏简介区域**：

- 位于名称下方
- 内容：3-4句话描述游戏世界观
- 字体：Inter，16px，浅灰色
- 排版：段落式，行高1.8
- 装饰：左侧一条竖直的青色发光线条作为引导

**视觉装饰**：

- 区域右侧边缘：垂直流动的数据流动画（类似代码雨但更抽象）
- 底部：半透明的星球轮廓或星图图案

### 2.3 区域2详情 — 用户名输入

**位置**：页面右侧（或移动端下方）

**输入表单结构**：

```
┌─────────────────────────────┐
│                             │
│   🛸  输入你的指挥官代号     │  ← 标签（Rajdhani 16px 青色）
│                             │
│   ┌─────────────────────┐   │
│   │  _________________  │   │  ← 输入框
│   └─────────────────────┘   │
│                             │
│   ┌─────────────────────┐   │
│   │    ▶ 开始游戏        │   │  ← 主按钮
│   └─────────────────────┘   │
│                             │
└─────────────────────────────┘
```

**输入框细节**：

- 宽度：最大320px，居中
- 高度：48px
- 内部左侧：小图标（指挥官头像占位符）
- 占位符文字：「请输入代号...」
- 最大长度：12个字符
- 输入时：每个字符出现时有打字机光标闪烁效果

**"开始游戏"按钮**：

- 尺寸：宽280px × 高52px
- 图标：左侧有播放/启动图标
- 默认状态：禁用态（灰色半透明），输入有效后激活
- 激活后：青色渐变背景 + 脉冲呼吸动画
- 点击后：按钮产生能量汇聚效果，然后整页过渡到下一页面

- 尺寸：宽500px × 自适应高度
- 边框：青色发光边框
- 关闭：点击按钮后淡出消失，不可通过其他方式关闭
- 动画：从底部滑入 + 缩放弹入

- 边框：默认半透明，悬停时对应角色颜色的发光边框
- 圆角：10px
- 悬停效果：轻微上浮（translateY: -4px）+ 边框发光增强
- 选中状态：边框变为青色强发光 + 左侧竖条高亮指示器

**头像区**：

- 形状：圆形或六边形（更具科幻感）
- 边框厚度：2px，颜色根据Agent角色类型分配
- 背景填充：角色代表色的半透明版本
- 如果没有实际头像：使用首字母/符号作为占位符

**基础变量展示**：

- 格式：图标 + 属性名 + 数值 + 进度条
- 进度条：细长条形，填充色根据数值高低从绿→黄→红渐变
- 数值字体：JetBrains Mono 14px

**特点描述**：

- 最多3行文字，超出部分省略号
- 悬停卡片时可展开查看完整描述（tooltip或展开模式）

**单个架构卡片设计**：

- 尺寸：宽100% × 高120px

- 默认状态：半透明边框 + 低亮度图标
- 悬停状态：边框发光 + 图标亮起 + 显示简要说明tooltip

- **网状架构**：网状互联的多节点图，每个节点连接多个其他节点

**确认按钮**：

- 仅在选择一个架构后激活
- 文字：「进入架构配置」或「确认此架构」
- 样式同主按钮（青色渐变）

### 4.3 左侧区域 — 架构图（可交互）

**功能需求**：

- 显示当前选择的架构类型的可视化图
- 用户可以添加/删除节点槽位
- 用户可以将底部卡片池中的Agent拖拽到架构图的某个位置上
- 可以调整已放置Agent的位置

**树形架构图具体设计**：

```
┌────────────────────────────────────┐
│  🌳 树形架构                        │  ← 区域标题
│                                    │
│            ┌──────┐                │
│            │ [根]  │                │  ← 根节点（可放置Agent）
│            └──┬───┘                │
│         ┌────┼────┐                │
│         ▼    ▼    ▼                │
│     ┌────┐┌────┐┌────┐            │  ← 第二层节点
│     │[ ] ││[ ] ││[ ] │            │     （空槽用虚线框表示）
│     └┬───┘└┬───┘└┬───┘            │
│    ┌┴┐   ┌┴┐   ┌┴┐               │  ← 第三层叶子节点
│    ▼▼    ▼▼    ▼▼                 │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐  │
│  │[]│ │[]│ │[]│ │[]│ │[]│ │[]│  │
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘  │
│                                    │
│  [+ 添加分支]  [- 删除选中节点]     │  ← 操作工具栏
└────────────────────────────────────┘
```

**节点样式**：

- **空槽位**：虚线边框圆角矩形，内部有"+"号或虚影人形图标，背景更深透明
- **已填充**：显示Agent的头像+名字小型卡片，带该Agent的代表色边框
- **连线**：从父节点到子节点的发光连线（青色半透明，粗细2px）
- **拖拽中**：被拖拽的卡片跟随鼠标/手指，目标槽位高亮闪烁

**操作工具栏**：

- 位于架构图底部
- 包含按钮：「添加节点」「删除节点」「清空重置」「自动排列」
- 按钮为小型次要按钮风格

### 4.4 右侧区域 — Agent实时聊天区（区域14）

## 六、动效与交互规范

### 6.1 全局动效原则

| 类型 | 时长 | 缓动函数 | 使用场景 |
|------|------|----------|----------|
| 微交互（hover） | 150ms | ease-out | 按钮悬停、卡片浮起 |
| 状态切换 | 250ms | ease-in-out | 展开/折叠、模态框 |
| 页面转场 | 400-600ms | ease-out | 页面间切换 |
| 数据更新 | 300ms | ease-out | 数值变化、进度条 |
| 强调动画 | 500-800ms | spring | 重要事件通知、成就解锁 |

### 6.2 特殊动效

**页面切换**：

- 从旧页面到新页面：旧页面向左淡出 + 新页面从右滑入
- 或者：旧页面缩小消失 + 新页面放大出现

**数值变化**：

- 数字变化时：滚动计数效果（从旧值滚动到新值）
- 正增长：短暂绿色闪光
- 负增长：短暂红色闪光 + 抖动

**Agent聊天新消息**：

- 消息从下方淡入 + 轻微向上滑动
- 被@的名字：脉冲高亮效果

**星辰值更新**：

- 数字放大闪烁 + 粒子喷发效果（增加时）

### 7.1 断点定义

| 断点 | 宽度范围 | 布局变化 |
|------|----------|----------|

**页面2（主选择页）**：

- 平板：上（背景+架构）+ 下（卡片）两栏

**页面3（架构决策页）**：

- 桌面：左（架构图40%）+ 右（聊天60%）+ 底部（卡片池）

- 手机：架构图 → 聊天 → 卡片池，三个可切换的标签页
