# 模块五：宏观变量与产出公式

> 本文档定义宏观变量的计算公式，包括能级、向心力、信息保真度等，以及最终产出的完整公式。

---

## 一、宏观变量总览

### 1.1 变量列表

| 变量 | 符号 | 范围 | 含义 |
|------|------|------|------|
| 能级 | E_level | [0, 1] | 文明整体能量/活跃状态 |
| 向心力 | C | [0, 1] | 文明凝聚力和一致度 |
| 信息保真度 | F | [0, 1] | 信息传播准确性 |
| 社会资本 | S | [0, 1] | 整体信任和协作水平 |
| 效率指数 | η_avg | [0, 1] | 平均转化效率 |

### 1.2 聚合原则

宏观变量由微观变量聚合而来：

```
Macro = Aggregation(Micro_1, Micro_2, ..., Micro_N)
```

聚合方法：
- **求和**：适用于累加型变量
- **平均**：适用于分布型变量
- **方差分析**：适用于一致性变量

## 二、能级（Energy Level）

### 2.1 定义

能级反映文明整体的活跃程度和产出能力。

### 2.2 计算公式

能级由Agent的体力状态和产出效率聚合：

```
E_level = (1/N) × Σ E_i_norm × η_i_norm
```

其中：
- N = Agent数量 = 10
- E_i_norm = Agent i 的剩余体力 / 100（归一化）
- η_i_norm = Agent i 的转化率（已归一化）

### 2.3 详细展开

```
E_level = (1/10) × Σ [E_i/100] × [η_i]
```

**示例计算**：

| Agent | 剩余体力 | E_i_norm | η_i | 贡献项 |
|-------|---------|----------|-----|--------|
| A1 | 60 | 0.60 | 0.45 | 0.27 |
| A2 | 45 | 0.45 | 0.38 | 0.17 |
| A3 | 80 | 0.80 | 0.52 | 0.42 |
| ... | ... | ... | ... | ... |
| 平均 | - | - | - | **0.32** |

E_level = 0.32

### 2.4 能级的意义

| 能级范围 | 文明状态 | 效果 |
|---------|---------|------|
| E_level > 0.7 | 高度活跃 | 产出加成 +10% |
| 0.5 < E_level ≤ 0.7 | 正常运行 | 无加成 |
| 0.3 < E_level ≤ 0.5 | 疲惫 | 产出惩罚 -10% |
| E_level ≤ 0.3 | 瘫痪 | 产出惩罚 -30% |

## 三、向心力（Centripetal Force）

### 3.1 定义

向心力衡量文明成员目标一致性和凝聚力。

### 3.2 基础公式

```
C = Average(L) × (1 - Variance(L)_normalized)
```

其中：
- L = 各Agent的忠诚度向量 [L_1, L_2, ..., L_N]
- Average(L) = 平均忠诚度
- Variance(L)_normalized = 归一化方差

### 3.3 详细推导

**平均忠诚度**：
```
L_avg = (1/N) × Σ L_i
```

**忠诚度方差**：
```
Var(L) = (1/N) × Σ (L_i - L_avg)²
```

**归一化方差**（调整后公式，参见评审报告问题1.3修复）：
```
Var_norm(L) = min(Var(L) / 0.1, 1)
```

**设计理由**：原公式使用0.25作为归一化分母，但实际游戏中忠诚度很少极端分化。调整为0.1使方差惩罚更敏感，能够反映"轻微分化"到"严重分裂"的区别。

### 3.4 计算示例

假设忠诚度向量 L = [0.8, 0.7, 0.9, 0.6, 0.8, 0.7, 0.9, 0.5, 0.8, 0.7]

```
L_avg = (0.8+0.7+0.9+0.6+0.8+0.7+0.9+0.5+0.8+0.7)/10 = 0.74

Var(L) = Σ(L_i - 0.74)² / 10
       = (0.0036 + 0.0016 + 0.0256 + 0.0196 + 0.0036 + 0.0016 + 0.0256 + 0.0576 + 0.0036 + 0.0016) / 10
       = 0.0144

Var_norm = min(0.0144 / 0.1, 1) = 0.144

C = 0.74 × (1 - 0.144) = 0.74 × 0.856 = 0.633
```

**对比原公式**：原公式 Var_norm = 0.0144/0.25 = 0.0576，C = 0.697
新公式更能反映忠诚度分化的影响。

### 3.5 向心力的意义

| 向心力范围 | 文明状态 | 效果 |
|-----------|---------|------|
| C > 0.8 | 高度团结 | 协作加成 +15% |
| 0.6 < C ≤ 0.8 | 基本团结 | 无加成 |
| 0.4 < C ≤ 0.6 | 分化 | 内耗惩罚 -10% |
| C ≤ 0.4 | 分裂 | 内耗惩罚 -25% |

## 四、信息保真度（Fidelity）

### 4.1 定义

信息保真度衡量文明内信息传播的准确性。

### 4.2 计算公式

```
F = Average(F_ij) for all communications in this cycle
```

其中 F_ij 是第 j 条通讯的保真系数（见模块四）。

### 4.3 与架构的关系

不同架构的信息保真度基础值不同：

| 架构 | 基础保真度 | 原因 |
|------|-----------|------|
| 星形 | 0.9 | 信息直达中心，转发少 |
| 树形 | 0.7 | 层级转发，累积失真 |
| 网状 | 0.85 | 多路径传播，互相校正 |
| 部落 | 0.6 | 跨部落通讯距离长 |

### 4.4 动态保真度

实际保真度受以下因素影响：

```
F_actual = F_base × Trust_factor × Entropy_factor
```

**信任因子**：
```
Trust_factor = Average(all T[i][j])
```

**认知熵因子**：
```
Entropy_factor = 1 - 0.3 × Average(H_i)
```

### 4.5 信息保真度的意义

| 保真度范围 | 信息状态 | 效果 |
|-----------|---------|------|
| F > 0.8 | 通讯清晰 | 决策质量 +10% |
| 0.6 < F ≤ 0.8 | 基本清晰 | 无加成 |
| 0.4 < F ≤ 0.6 | 有噪音 | 误判概率增加 |
| F ≤ 0.4 | 通讯混乱 | 决策质量 -20% |

## 五、社会资本（Social Capital）

### 5.1 定义

社会资本衡量文明整体的信任和协作网络强度。

### 5.2 计算公式

```
S = Average(T[i][j]) for all i≠j
     + Density_factor × Edge_density
```

**信任平均值**：
```
Trust_avg = (1 / (N×(N-1))) × Σ Σ T[i][j]  for i≠j
```

**密度因子**：
```
Density_factor = 0.2
Edge_density = 实际边数 / 最大可能边数
```

### 5.3 计算示例

假设信任矩阵平均值为 0.65，网状架构（Edge_density=1）：

```
S = 0.65 + 0.2 × 1.0 = 0.85
```

部落架构（Edge_density=0.33）：

```
S = 0.65 + 0.2 × 0.33 = 0.72
```

### 5.4 社会资本的意义

| 社会资本范围 | 协作状态 | 效果 |
|-------------|---------|------|
| S > 0.8 | 高度协作 | 协作产出 +20% |
| 0.6 < S ≤ 0.8 | 正常协作 | 无加成 |
| 0.4 < S ≤ 0.6 | 协作不足 | 孤立惩罚 -10% |
| S ≤ 0.4 | 原子化 | 严重孤立 -25% |

## 六、效率指数（Efficiency Index）

### 6.1 定义

效率指数是所有Agent转化率的平均值。

### 6.2 计算公式

```
η_avg = (1/N) × Σ η_i
```

各Agent的η_i 计算见模块二。

### 6.3 与架构的关系

```
η_avg_expected = f(architecture, personality_distribution)
```

不同架构对效率的影响：

| 架构 | 效率预期 | 原因 |
|------|---------|------|
| 星形 | 高（中心高效） | 核心节点高效，边缘依赖核心 |
| 树形 | 中等 | 层级传递有损耗 |
| 网状 | 变化大 | 依赖个体能力和社会资本 |
| 部落 | 分化 | 部落内高，部落间低 |

## 七、最终产出公式

### 7.1 单循环产出

```
P_cycle = Σ (E_work_i × η_i) × Modifiers × ArchitectureEfficiency
```

其中 Modifiers 是宏观变量修正系数：

```
Modifiers = Energy_factor × Cohesion_factor × Fidelity_factor × Social_factor
```

### 7.2 各因子定义

| 因子 | 公式 | 基于变量 |
|------|------|---------|
| Energy_factor | 0.7 + 0.3 × E_level | 能级 |
| Cohesion_factor | 0.5 + 0.5 × C | 向心力 |
| Fidelity_factor | 0.8 + 0.2 × F | 信息保真度 |
| Social_factor | 0.7 + 0.3 × S | 社会资本 |

### 7.3 架构效率系数（评审报告问题3.1修复）

为平衡不同架构的风险收益，引入架构效率系数：

```
ArchitectureEfficiency = BaseEfficiency × PositionBonus
```

**基础效率**：

| 架构 | BaseEfficiency | 说明 |
|------|---------------|------|
| 星形 | 1.3 | 高风险高回报，补偿易崩溃风险 |
| 树形 | 1.0 | 标准效率 |
| 网状 | 1.0 | 高鲁棒性已是优势 |
| 部落 | 1.1 | 中等难度补偿 |

**位置加成**（针对星形架构）：

核心节点产出加成：
```
PositionBonus_core = 1.5  // 星形架构核心节点额外50%
```

### 7.4 完整公式

```
P_cycle = Σ_i [E_work_i × η_i × PositionBonus_i]
          × (0.7 + 0.3 × E_level)
          × (0.5 + 0.5 × C)
          × (0.8 + 0.2 × F)
          × (0.7 + 0.3 × S)
          × ArchitectureEfficiency
```

### 7.5 公式简化

设基础产出为：

```
P_base = Σ_i [E_work_i × η_i × PositionBonus_i]
```

则：

```
P_cycle = P_base × K_energy × K_cohesion × K_fidelity × K_social × K_arch
```

各系数范围：

| 系数 | 范围 | 典型值 |
|------|------|--------|
| K_energy | [0.7, 1.0] | 0.85 |
| K_cohesion | [0.5, 1.0] | 0.75 |
| K_fidelity | [0.8, 1.0] | 0.90 |
| K_social | [0.7, 1.0] | 0.80 |
| K_arch | [1.0, 1.3] | 1.0 |

综合修正系数范围：[0.196, 1.56]（含架构系数）

### 7.6 架构平衡说明

**星形架构平衡**：
- 循环数少（5循环 vs 网状10循环）
- 但每循环效率高（K_arch = 1.3 + 核心节点1.5倍）
- 核心内鬼风险大（鲁棒性=0.1）

**网状架构平衡**：
- 循环数多（10循环）
- 效率标准（K_arch = 1.0）
- 鲁棒性高（0.9），内鬼威胁小

这样形成了"高风险高回报"与"稳定低风险"的架构选择策略。

### 7.5 归一化产出

为便于比较，产出可归一化：

```
P_normalized = P_cycle / P_max_expected
```

其中 P_max_expected = N × E_work_max × η_max × 1.0

```
P_max_expected = 10 × 80 × 1.0 = 800
```

## 八、多回合累计

### 8.1 累计产出

```
P_total = Σ P_cycle for all cycles in all rounds
```

### 8.2 时间衰减（可选）

较早的产出价值略低（模拟折旧）：

```
P_total = Σ P_cycle × decay_factor^age
```

其中 decay_factor = 0.95，age = 回合差

### 8.3 最终得分

```
Score = P_total
```

**架构奖励系数**（评审报告问题3.3修复）：

所有架构奖励系数统一为1.0，让架构优劣势自然体现在游戏机制中，而非人为调整。

| 架构 | Architecture_bonus | 说明 |
|------|-------------------|------|
| 星形 | 1.0 | 风险收益已在效率系数中体现 |
| 树形 | 1.0 | 标准架构 |
| 网状 | 1.0 | 高鲁棒性已是优势 |
| 部落 | 1.0 | 不额外奖励 |

## 九、胜负判定

### 9.1 比较规则

三个文明按 Score 排序：

```
Rank_1: Civilization with highest Score
Rank_2: Civilization with second highest Score
Rank_3: Civilization with lowest Score
```

### 9.2 特殊结束条件

| 条件 | 结果 |
|------|------|
| 所有Agent叛变 | 文明崩溃，Score = 0 |
| 核心节点死亡（星形） | Score × 0.1 |
| 社会资本 < 0.2 | Score × 0.5 |
