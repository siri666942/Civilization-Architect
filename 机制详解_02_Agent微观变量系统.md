## 二、体力值系统（Energy）

### 2.1 基本定义

| 属性 | 值 |
|------|-----|
| 符号 | E |
| 范围 | [0, 100] |
| 重置方案 | 循环级重置（每循环开始恢复到100） |
| 消耗分配 | 干活（产出）、内斗（斗争）、通讯（交流） |

### 2.2 体力分配机制

Agent在每个循环中需要决定体力如何分配：

```
E_work + E_conflict + E_comm = E_total ≤ 100
```

**分配决策由AI自主决定**，但受性格属性影响：

| 性格属性 | 对分配的影响 |
|---------|-------------|
| Authority↑ | 更多E_comm（向上汇报） |
| Selfishness↑ | 更多E_work（个人产出）或E_conflict（权力斗争） |
| Altruism↑ | 更多E_work（为集体产出） |
| LoyaltyBase↓ | 可能更多E_conflict（内斗） |

### 2.3 AI调用优化（评审报告问题4.1修复）

**优化策略**：体力分配每回合决策一次，而非每循环。

| 决策类型 | 频率 | 原因 |
|---------|------|------|
| 体力分配 | 每回合1次 | 减少AI调用，策略保持稳定 |
| 内鬼行为 | 每循环判断 | 核心玩法，需要动态响应 |
| 通讯内容 | 按需生成 | 简单场景用模板 |

**AI调用估算**（优化后）：

```
网状架构（10循环/回合）：
- 体力分配：10 Agent × 1次 = 10次/回合
- 内鬼行为：10 Agent × 10循环 = 100次/回合（仅内鬼判断）
- 通讯内容：约30%调用AI = 30次/回合
- 总计：约140次/回合（原300次）

3文明并行：约420次/回合（原900次）
10回合游戏：约4200次（原9000次）
```

### 2.4 AI决策的边界约束

AI在决定体力分配时，需在以下边界内：

```
E_work ∈ [20, 80]      // 至少20%用于产出，最多80%
E_conflict ∈ [0, 40]   // 内斗最多40%
E_comm ∈ [5, 30]       // 通讯至少5%，最多30%
```

### 2.4 AI决策提示词模板

```
你现在需要决定本循环的体力分配。你有100点体力。

你的性格：{personality}

可选行动：
1. 干活（E_work）：产出星尘，贡献给文明
2. 内斗（E_conflict）：消耗他人体力，权力斗争
3. 通讯（E_comm）：信息传递，建立关系

约束条件：
- E_work: [20, 80]
- E_conflict: [0, 40]
- E_comm: [5, 30]
- 三者之和 ≤ 100

请输出JSON格式的分配决策：
{
  "E_work": number,
  "E_conflict": number,
  "E_comm": number,
  "reasoning": "简要说明分配理由"
}
```

### 2.5 体力消耗规则

| 行为 | 消耗公式 | 说明 |
|------|---------|------|
| 干活 | ΔE = -E_work | 直接消耗 |
| 内斗（发起方） | ΔE = -E_conflict | 发起方消耗 |
| 内斗（被攻击方） | ΔE = -0.3 × E_conflict_received | 被动损失 |
| 通讯（发送） | ΔE = -E_comm | 发送方消耗 |
| 通讯（接收） | ΔE = -0.1 × E_comm_received | 接收处理消耗 |

## 三、认知熵系统（Cognitive Entropy）

### 3.1 定义

**认知熵 H** 衡量Agent的思维混乱程度：
- H = 0：思维完全清晰，决策质量最高
- H = 1：极度混乱，决策可能完全随机

**理论基础**：信息熵理论，参考Shannon熵概念

### 3.2 认知熵的来源

认知熵由**累积压力**产生：

```
H = 1 - e^(-Σ stress_factors)
```

### 3.3 认知熵增长因素

| 触发因素 | 增量公式 | 理论依据 |
|---------|---------|---------|
| 信息过载 | ΔH += 0.05 × (消息数 - 处理能力) | 认知负荷理论 |
| 冲突体验 | ΔH += 0.03 × 内斗次数 | 压力累积 |
| 忠诚度撕裂 | ΔH += 0.02 × \|上级指令 - 个人意愿\| | 认知失调理论 |
| 被欺骗 | ΔH += 0.08 × 发现欺骗次数 | 信任崩塌 |
| 架构层级 | ΔH += 0.01 × depth（在树形架构中） | 信息失真累积 |

### 3.4 认知熵衰减因素

| 恢复因素 | 衰减公式 | 理论依据 |
|---------|---------|---------|
| 休息周期 | ΔH -= 0.02 × (1 - H) | 自然恢复 |
| 韧性属性 | ΔH ×= (1 - 0.5 × Resilience) | 心理弹性 |
| 同伴支持 | ΔH -= 0.03 × support_intensity | 社会支持理论 |

### 3.5 认知熵完整更新公式

```
H(t+1) = H(t)
         + ΔH_info_overload
         + ΔH_conflict
         + ΔH_dissonance
         + ΔH_deception
         + ΔH_hierarchy
         - ΔH_rest
         - ΔH_support

# 应用韧性衰减
H(t+1) = H(t+1) × (1 - 0.5 × Resilience)

# 边界约束
H(t+1) = clamp(H(t+1), 0, 1)
```

### 3.6 认知熵对行为的影响

| 认知熵范围 | 影响效果 |
|-----------|---------|
| H < 0.3 | 无影响，决策正常 |
| 0.3 ≤ H < 0.6 | 决策质量下降10%，可能误判 |
| 0.6 ≤ H < 0.8 | 决策质量下降30%，易被误导 |
| H ≥ 0.8 | 决策质量下降50%，行为接近随机 |

**数学表达**：

```
DecisionQuality = Intelligence × (1 - 0.5 × H²)
```

当 H=0.6 时，决策质量衰减为 Intelligence × 0.82
当 H=0.8 时，决策质量衰减为 Intelligence × 0.68

## 四、忠诚度系统（Loyalty）

### 4.1 定义

**当前忠诚度 L** 是动态变化的，初始值为忠诚度基准（LoyaltyBase），但会随环境变化。

### 4.2 忠诚度变化因素

| 触发因素 | 变化公式 | 方向 |
|---------|---------|------|
| 组织成功 | ΔL += 0.05 × success_degree | ↑ |
| 组织失败 | ΔL -= 0.05 × failure_degree | ↓ |
| 个人收益被忽视 | ΔL -= 0.03 × 自私需求未满足 | ↓ |
| 集体认可 | ΔL += 0.02 × recognition_count | ↑ |
| 被内鬼诱导 | ΔL -= 0.04 × 诱导强度 | ↓ |
| 发现内鬼 | ΔL += 0.02 × 发现奖励 | ↑ |

### 4.3 忠诚度完整更新公式

```
L(t+1) = L(t) + ΔL_organizational_success
                 + ΔL_organizational_failure
                 + ΔL_personal_grievance
                 + ΔL_recognition
                 - ΔL_manipulation
                 + ΔL_traitor_discovery

# 向基准值回归（避免长期偏离）
L(t+1) = 0.9 × L(t+1) + 0.1 × LoyaltyBase

# 边界约束
L(t+1) = clamp(L(t+1), 0, 1)
```

### 4.4 忠诚度阈值效应

| 忠诚度范围 | 行为倾向 | 触发条件 |
|-----------|---------|---------|
| L > 0.8 | 完全合作 | 正常工作 |
| 0.5 < L ≤ 0.8 | 有所保留 | 可能摸鱼 |
| 0.3 < L ≤ 0.5 | 消极抵抗 | 大幅摸鱼 |
| L ≤ 0.3 | 叛变风险 | 可能成为内鬼 |

### 4.5 忠诚度对转化率的影响

```
η_loyalty = 0.5 + 0.5 × L
```

- L=1时，η_loyalty = 1.0（完全投入）
- L=0.5时，η_loyalty = 0.75（有所保留）
- L=0时，η_loyalty = 0.5（消极对待）

## 五、信任矩阵系统（Trust Matrix）

### 5.1 定义

**信任矩阵 T** 是一个 N×N 的矩阵（N=10），T[i][j] 表示Agent i 对 Agent j 的信任度。

```
T = [
  [1.00, 0.50, 0.50, ...],  // A1对所有人的信任
  [0.50, 1.00, 0.50, ...],  // A2对所有人的信任
  ...
]
```

**规则**：
- T[i][i] = 1.0（对自己的信任）
- 初始时 T[i][j] = 0.5（中性信任）

### 5.2 信任度更新机制

基于**社会认知理论（Social Cognitive Theory）**，信任度通过交互更新：

```
T[i][j](t+1) = T[i][j](t) × (1 - γ_decay) + α × (observed_behavior - expected_behavior)
```

其中：
- γ_decay = 0.02（衰减率，参见评审报告问题1.5修复）
- α = 0.1 ~ 0.2（学习率）
- observed_behavior：观察到的行为（合作=1，背叛=0）
- expected_behavior = T[i][j](t)（基于当前信任的期望）

**设计理由**（评审报告问题1.5修复）：

引入衰减机制防止信任值无限累积趋向边界，保持动态性：
- 无衰减时，高激活Agent间信任持续增加，最终趋向1.0
- 有衰减时，信任会逐渐"遗忘"，需要持续正向交互维持

**展开**：
```
# 正向交互（合作）
T[i][j](t+1) = T[i][j](t) × 0.98 + 0.1 × (1 - T[i][j](t))

# 负向交互（背叛）
T[i][j](t+1) = T[i][j](t) × 0.98 - 0.2 × T[i][j](t))

# 无交互时（自然衰减）
T[i][j](t+1) = T[i][j](t) × 0.98  // 轻微向中性值0.5回归
```

**注意**：负面事件的影响是正面事件的2倍（负面偏差，Negativity Bias）

### 5.3 信任传播机制

基于**信任传递理论**，A信任B，B信任C → A间接信任C：

```
T[i][k]_indirect = T[i][j] × T[j][k]
```

综合信任：
```
T[i][k]_total = w_direct × T[i][k] + w_indirect × T[i][k]_indirect
```

其中：
- w_direct = 0.7，w_indirect = 0.3

### 5.4 社交性对信任建立的影响

高社交性（Sociability）的Agent更快建立信任：

```
α_actual = α_base + 0.1 × Sociability
```

- Sociability=1的Agent，学习率=0.2（快速建立/破坏信任）
- Sociability=0的Agent，学习率=0.1（慢速建立信任）

### 5.5 信任矩阵的性质

信任矩阵实际上是**神经网络中的权重矩阵**：
- 信息传播时：Message(i→j) = Content × T[i][j]
- 低信任度导致信息被削弱或忽略

## 六、转化率系统（Efficiency）

### 6.1 定义

**转化率 η** 表示Agent投入的体力能转化为实际产出的效率：

```
ActualOutput = E_work × η
```

### 6.2 转化率计算公式

转化率受多个因素影响：

```
η = η_base × η_intelligence × η_loyalty × η_entropy × η_trust
```

| 因子 | 公式 | 说明 |
|------|------|------|
| η_base | 0.8 | 基础转化率 |
| η_intelligence | 0.8 + 0.2 × Intelligence | 智力加成 |
| η_loyalty | 0.5 + 0.5 × L | 忠诚度加成 |
| η_entropy | 1 - 0.5 × H | 认知熵惩罚 |
| η_trust | 社会资本因子（见下） | 信任环境 |

### 6.3 社会资本因子

社会资本因子反映Agent所处的社会支持环境：

```
η_trust = Average(T[j][i]) for all j connected to i
```

即：所有与i有连接的Agent对i的信任平均值。

- 被信任的Agent更容易获得资源和信息，效率更高
- 被孤立的Agent效率较低

### 6.4 完整转化率公式

```
η = 0.8 × (0.8 + 0.2 × Intelligence)
    × (0.5 + 0.5 × L)
    × (1 - 0.5 × H)
    × Average(T[connected_agents][self])
```

**数值范围**：η ∈ [0.16, 1.0]

**示例计算**：
- Intelligence=0.8, L=0.7, H=0.2, AverageTrust=0.6
- η = 0.8 × 0.96 × 0.85 × 0.9 × 0.6 = 0.35

## 七、贡献值系统（Production）

### 7.1 定义

**贡献值 P** 是Agent累计产出星尘的度量。

### 7.2 单循环贡献计算

```
P_single_cycle = E_work × η × (1 + Random_factor)
```

其中 Random_factor ∈ [-0.1, 0.1]，模拟不确定因素。

### 7.3 累计贡献

```
P_total = Σ P_single_cycle
```

### 7.4 贡献值的意义

- 用于计算能级（宏观变量）
- 用于识别高产出/低产出Agent
- 可能影响Agent的职位/地位（后续版本）

## 八、心理负担系统（Mental Load）

### 8.1 定义

心理负担是认知熵、忠诚度冲突、社会压力的综合指标，用于驱动Agent的行为倾向。

### 8.2 计算公式

```
MentalLoad = H
             + 0.3 × (1 - L)
             + 0.2 × (1 - AverageTrust)
             + 0.1 × Position_Stress
```

其中：
- H：认知熵
- (1 - L)：忠诚度缺失
- (1 - AverageTrust)：社会孤立程度
- Position_Stress：职位压力（核心职位压力大）

### 8.3 职位压力

不同架构位置的Agent有不同的压力：

| 位置 | Position_Stress |
|------|----------------|
| 核心（星形中心） | 0.5 |
| 中层管理者 | 0.3 |
| 边缘执行者 | 0.1 |

### 8.4 心理负担阈值

| MentalLoad | 状态 | 行为倾向 |
|-----------|------|---------|
| < 0.4 | 轻松 | 正常工作 |
| 0.4 - 0.6 | 压力 | 可能寻求支持 |
| 0.6 - 0.8 | 过载 | 效率下降，可能摸鱼 |
| > 0.8 | 崩溃边缘 | 可能叛变或退出 |

## 九、变量间的交互关系总结

### 9.1 正反馈循环

```
信任高 → 社会资本高 → 效率高 → 产出高 → 被认可 → 忠诚度↑ → 信任高
```

### 9.2 负反馈循环

```
信任低 → 社会资本低 → 效率低 → 产出低 → 被忽视 → 忠诚度↓ → 内斗↑ → 信任低
```

### 9.3 认知熵的放大效应

```
认知熵↑ → 决策质量↓ → 误判↑ → 信任受损 → 社会孤立 → 认知熵↑↑
```

### 9.4 韧性的缓冲作用

```
韧性高 → 认知熵衰减快 → 恢复能力强 → 抗压能力高
```
