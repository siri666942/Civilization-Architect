## 八、模拟测试发现的新问题

### 问题8.1：循环效率补偿机制未实际应用

**问题描述**：
机制文档定义了循环效率补偿公式：
```
CycleEfficiencyBonus = 1 + 0.5 × (1 - R_normalized)
```
但在产出计算中未应用此系数。

**问题影响**：
- 网状架构（10循环）与星形架构（5循环）产出差距仍为2倍
- 低通达度架构未获得足够补偿

**改进方案**：
在产出公式中添加循环效率补偿：
```
P_cycle = P_base × K_energy × K_cohesion × K_fidelity × K_social × K_arch × CycleEfficiencyBonus
```

### 问题8.2：向心力方差归一化过于敏感

**问题描述**：
当前公式：`C = 平均忠诚度 × (1 - min(Var(L)/0.1, 1))`

实际测试：
- 网状架构忠诚度方差 = 0.36
- 归一化后 = min(0.36/0.1, 1) = 1.0
- 向心力 = 0.49 × 0 = 0

**问题原因**：
分母0.1过于敏感，当方差超过0.1时向心力直接归零。

**改进方案**：
调整归一化公式为二次函数衰减：
```
C = 平均忠诚度 × (1 - min(Var(L)/0.1, 1))²
```
或使用更温和的分母：
```
C = 平均忠诚度 × (1 - min(Var(L)/0.25, 1))
```

### 问题8.3：保真度计算初始值偏低

**问题描述**：
所有架构的最终保真度 < 0.5，主要原因是：
1. 初始信任值 = 0.5（中性）
2. 基础保真度 × 信任因子 × 熵因子 导致累积衰减

**改进方案**：
1. 提高初始信任值：
   ```
   T[i][j]_init = 0.6 + 0.1 × (社交性 - 0.5)
   ```
2. 在保真度计算中添加最小值：
   ```
   F = max(0.5, F_base × Trust_factor × Entropy_factor)
   ```

---

### 问题8.4：内鬼激活阈值过高

**问题描述**：
模拟中所有文明的高内鬼倾向Agent数量为2-3个，但活跃内鬼为0。

**问题原因**：
激活条件：
```
P_activate = TraitorTendency × (1 - L_current) × OpportunityFactor
if P_activate > 0.5:
    激活
```
条件过于严格。

**改进方案**：
1. 降低激活阈值或采用累积概率
2. 增加随机激活事件
